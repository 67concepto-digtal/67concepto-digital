**Última actualización:** 24 de Julio 2026

---

## 📋 OBJETIVO

Plantilla estándar para realizar diagnóstico profesional a potenciales clientes.

Sirve para:
- Entender realmente sus problemas
- Cuantificar impacto económico
- Diseñar solución personalizada
- Generar propuesta comercial

---

## 🎯 ESTRUCTURA DE DIAGNÓSTICO (90 minutos)

### FASE 1: APERTURA (10 minutos)

**Bienvenida profesional:**
```
"Hola [Nombre], gracias por tu tiempo.

Nuestro objetivo en estos 90 minutos es:
1. Entender tu negocio profundamente
2. Identificar tus principales retos
3. Ver si podemos ayudarte realmente
4. Si es así, diseñar una solución personalizada

¿Te parece bien si comenzamos?"
```

**Confidencialidad:**
```
"Todo lo que converses hoy es confidencial.
No compartimos información con nadie.
Solo usamos esto para tu propuesta."
```

---

### FASE 2: CONTEXTO DEL NEGOCIO (15 minutos)

**Preguntas clave:**

```
1. DATOS BÁSICOS:
   ├─ "¿Cuánto tiempo llevas con este negocio?"
   ├─ "¿Cuántas personas trabajan?"
   ├─ "¿En qué ciudades/localidades operan?"
   └─ "¿Cuál es tu ingreso promedio mensual?"

2. MODELO DE NEGOCIO:
   ├─ "¿Cómo adquieres clientes?"
   ├─ "¿Cuál es tu margen de ganancia?"
   ├─ "¿Quiénes son tus principales competidores?"
   └─ "¿Qué te diferencia de ellos?"

3. TENDENCIAS:
   ├─ "¿Cómo van las ventas comparado con hace 1 año?"
   ├─ "¿Qué cambios ves en tus clientes?"
   ├─ "¿Cuál es tu plan para los próximos 12 meses?"
   └─ "¿Dónde ves tu negocio en 3 años?"

NOTA: Escuchar más que preguntar. Tomar notas.
```

---

### FASE 3: PROBLEMAS ACTUALES (25 minutos)

**Preguntas abiertas:**

```
PROBLEMA GENERAL:
"¿Cuál es tu principal frustración con tu negocio hoy?"
(Escuchar 2-3 minutos sin interrumpir)

PROFUNDIZAR EN PROBLEMA 1:
├─ "¿Me das un ejemplo específico?"
├─ "¿Con qué frecuencia pasa esto?"
├─ "¿Quién más se ve afectado?"
└─ "¿Cuál es el costo de esto para ti?"

PROBLEMAS SECUNDARIOS:
"¿Hay otros retos que enfrentes?"
(Dejar hablar 2-3 minutos)

PROFUNDIZAR EN PROBLEMA 2:
├─ "¿Cómo impacta esto en tu día a día?"
├─ "¿Ya intentaste resolver esto?"
└─ "¿Por qué no funcionó?"

PROBLEMAS TERCIARIOS:
"¿Hay algo más que no haya preguntado?"

CUANTIFICACIÓN:
Para cada problema:
├─ "¿Cuánto tiempo/dinero pierdes por esto?"
├─ "¿Si lo resolvieras, qué pasaría?"
└─ "¿Cuánto estarías dispuesto a invertir?"

MAPA DE PROBLEMAS (tomar notas):
Problema 1: [Desc] | Impacto: $[X]/mes | Frecuencia: [X]
Problema 2: [Desc] | Impacto: $[X]/mes | Frecuencia: [X]
Problema 3: [Desc] | Impacto: $[X]/mes | Frecuencia: [X]
```

---

### FASE 4: SISTEMAS ACTUALES (15 minutos)

**Entender qué tienen hoy:**

```
TECNOLOGÍA:
├─ "¿Qué sistemas usas actualmente?"
├─ "¿Cuánto tiempo llevas usándolos?"
├─ "¿Qué te gusta y qué no?"
└─ "¿Qué funciona bien y qué no?"

PROCESOS:
├─ "¿Cómo registras ventas?"
├─ "¿Cómo gestionar inventario?"
├─ "¿Cómo comunicas con clientes?"
├─ "¿Cómo tomas decisiones?"
└─ "¿Quién es responsable de cada área?"

DATOS:
├─ "¿Tienes acceso a tus números?"
├─ "¿Con qué frecuencia revisas resultados?"
├─ "¿Qué métricas monitoreas?"
└─ "¿Cómo predices tendencias?"

PERSONAL:
├─ "¿Tu equipo está capacitado en tech?"
├─ "¿Tienen ganas de aprender?"
└─ "¿Hay resistencia al cambio?"
```

---

### FASE 5: SOLUCIONES PREVIAS (10 minutos)

**Entender intentos anteriores:**

```
HISTORIA:
├─ "¿Qué soluciones has intentado?"
├─ "¿Cuánto invertiste?"
├─ "¿Por qué no funcionó?"
└─ "¿Qué aprendiste?"

EXPECTATIVAS:
├─ "¿Qué buscas en una solución?"
├─ "¿Cuál sería la solución ideal?"
├─ "¿Cuáles son tus requisitos mínimos?"
└─ "¿Hay features específicas que necesites?"

PRESUPUESTO:
├─ "¿Tienes presupuesto para resolver esto?"
├─ "¿Cuál sería tu inversión máxima?"
├─ "¿Prefieres pago único o mensual?"
└─ "¿En cuánto tiempo necesitas ROI?"

DECISIÓN:
├─ "¿Quién más necesita aprobar esto?"
├─ "¿Cuál es tu timeline para decidir?"
└─ "¿Qué haría que digas 'sí' definitivamente?"
```

---

### FASE 6: PRESENTACIÓN PRELIMINAR (10 minutos)

**Proponer dirección:**

```
SÍNTESIS:
"Basado en lo que conversos, entiendo que:

1. Tu principal desafío es: [Problema 1]
   └─ Costo: $[X]/mes

2. Secundariamente: [Problema 2]
   └─ Costo: $[X]/mes

3. Necesitas: [Requisito principal]

¿Es correcto? ¿Agregas algo?"

SOLUCIÓN PROPUESTA:
"Proponemos implementar:
├─ App 1 para resolver [Problema 1]
├─ App 2 para resolver [Problema 2]
├─ Integración con [Sistema actual]
└─ Timeline: 3-4 semanas

¿Te late así o le cambiaría algo?"

ROI PRELIMINAR:
"Si logramos esto, podrías:
├─ Ahorrar: $[X]/mes
├─ Ganar: $[X]/mes
└─ Total beneficio: $[X]/mes

Inversión: $[X]/mes
ROI: [X]%
Payback: [X] días

¿Te parece atractivo?"
```

---

### FASE 7: CIERRE (5 minutos)

**Siguiente paso claro:**

```
PROPUESTA:
"Lo que haría ahora es:

1. Consolidar este diagnóstico
2. Calcular ROI exacto
3. Diseñar plan de implementación
4. Presentarte propuesta formal en 3 días

¿Te parece bien?"

CONFIRMACIÓN:
├─ "¿Tienes dudas en este momento?"
├─ "¿Hay algo que deba aclarar?"
└─ "¿Es el mejor email/WhatsApp para contactarte?"

FOLLOW-UP:
"Te enviaré:
├─ Resumen de diagnóstico
├─ Propuesta comercial
├─ Links de calendarios para seguimiento
└─ Mi contacto directo"

DESPEDIDA:
"Gracias por tu tiempo y confianza.
Vamos a ayudarte a crecer tu negocio.
Hablaremos en 3 días."
```
